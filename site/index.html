Why teams switch from thefuck

oops is built for speed, portability, and zero-runtime dependency installs. It feels native on every platform and makes onboarding effortless.

10× faster startup

Rust binary starts in ~30ms vs ~300ms for Python-based alternatives. That means instant fixes during live demos.

One binary, no runtime

No Python, pip, or venv required. Download once, add your alias, and every shell gets corrections.

Cross-platform by default

Linux, macOS, and Windows are all first-class. Same commands, same experience, same fixes.

thefuck-compatible config

Environment variables and alias behaviour match what your team already uses, so migration is painless.

Install in one line

Use the installer scripts for each platform. They download the latest release and set up your path automatically.

macOS

Intel + Apple Silicon. Installs to /usr/local/bin.

curl -fsSL https://raw.githubusercontent.com/animeshkundu/oops/master/scripts/install.sh | bash

Linux

Supports x86_64 and ARM64 binaries.

curl -fsSL https://raw.githubusercontent.com/animeshkundu/oops/master/scripts/install.sh | bash

Windows (PowerShell)

Downloads the latest release and updates PATH.

iwr -useb https://raw.githubusercontent.com/animeshkundu/oops/master/scripts/install.ps1 | iex

Onboard in 60 seconds

New users can get to their first fix with four steps.

1

Install the binary

Use the one-line script above or your package manager.

2

Add the shell alias

Run eval "$(oops --alias)" once in your shell config.

3

Trigger a typo

Type git statsu and see oops suggest the fix.

4

Ship confidently

Use oops -y to auto-run the top fix.
