Configuration overview

oops ships with sane defaults, but power users can control rules, confirmations, and timeouts through a TOML file or env vars.

Config file

Create ~/.config/oops/config.toml to persist settings.

rules = ["ALL"]
exclude_rules = ["git_push_force"]
require_confirmation = true

Environment variables

Override settings quickly per-session.

export THEFUCK_REQUIRE_CONFIRMATION=false
export THEFUCK_RULES="sudo:git_push"

Common settings

Start with these values for a fast, safe setup.

  • require_confirmation: keep true for safety.
  • num_close_matches: raise to 5 for more suggestions.
  • slow_commands: add tools like gradle or docker.
